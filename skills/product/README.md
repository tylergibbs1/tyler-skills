# Product

Product-method skills for deciding what to build and in what order. Both model-invoked.

- **[proving-value-before-building-out](./proving-value-before-building-out/SKILL.md)**: Validate the core value headless, with an eval harness on real inputs, before any UI, services, or scale. Positive sequencing, distinct from scope-discipline.
- **[researching-latent-demand](./researching-latent-demand/SKILL.md)**: Decide what to build from real pain and market research, the painkiller-not-vitamin test, and competitor gap-mapping, then file the results as tracked issues.
